## Codebase Patterns
- Models live in `app/backend/src/db/models/` — UUID PKs use `server_default=func.gen_random_uuid()`, timestamps use `DateTime(timezone=True)` with `server_default=func.now()`
- All models use SQLAlchemy `Mapped` + `mapped_column` typed annotations (not classic Column style)
- The `.venv` in `app/backend/` was created with uv and pointed to Python 3.13 (now unavailable); migrations must be written manually or the venv must be recreated
- `alembic/env.py` must import every model explicitly (e.g. `from src.db.models import Guest, User, Wedding  # noqa: F401`) for autogenerate to see the table
- For back_populates between models, use `TYPE_CHECKING` guard in wedding.py to avoid circular imports; in guest.py use string `"Wedding"` for the relationship target
- Migration file naming: `YYYYmmdd_HHMM_<12-hex-rev>_<slug>.py`; down_revision must chain to the previous revision ID
